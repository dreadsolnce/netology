# Выгоды

1. Позволит использовать различный наиболее подходящий стек технологий для конкретной бизнес задачи.
2. Масштабируемость. Позволяет увеличивать экземпляры конкретных сервисов в зависимости от нагрузки на конкретные сервисы например в пиковые часы, и так же снижать количество экземпляров сервиса в в период когда нагрузка не большая. Экономия общего количества ресурсов.
3. Простота развертывания. Микросервисная архитектура позволяет развертывать отдельный сервис не затрагивая остальные, что существенно влияет на качество. Развертывание становится чаще и надежней, и при возникновении проблем небольшое изменение проще откатить. Получаем более высокую надежность всей системы.
4. Новая функциональность попадает пользователю гораздо быстрее.
5. Каждый отдельный сервис может быть реализован на своем языке программирования.
6. Микросервисы хорошо подходят для разработки крупных, сложных приложений

# Проблемы
1. Микросервисы накладывают повышенные требования к командам разработки и эксплуатации.
2. Следует много времени уделять прямой и обратной совместимости API
3. Нельзя допускать зависимость сервисов друг от друга
4. Необходимо как можно больше автоматизировать процесс сборки и тестирования.
5. Нужно иметь актуальную документацию
6. Нужные шаблоны и скрипты быстрого создания сервиса
7. Проблемы эксплуатации из за большого количества работающих сервисов, за которыми нужно следить и управлять.
8. Необходима единая система сбора и анализа логов.
